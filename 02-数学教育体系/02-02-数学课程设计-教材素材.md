## 1. 课程设计模板与清单

### 1.1 单元教学设计模板（Unit Design Template）

| **项目** | **说明与填写指南** |
| :--- | :--- |
| **单元主题** | 例如：圆的认识，一元二次方程，数据的分析 |
| **适用年级** |  |
| **时间安排** | 计划多少课时完成 |
| **核心概念** | 本单元学生必须理解的1-3个最重要的数学概念。 |
| **学习目标** | **知识与技能**：学生将能够...（用行为动词描述，如计算、证明、绘制）<br>**思维与过程**：学生将能够...（如分析、建模、论证）<br>**情感与态度**：学生将体会到...（如数学的严谨性、应用价值） |
| **核心问题** | 一个或多个开放性的、能够驱动整个单元学习的"大问题"。例如："我们如何用数学来做出更明智的决策？" |
| **评价计划** | **形成性评价**：课堂提问、小组讨论观察、练习题分析、出题卡...<br>**总结性评价**：单元测验、项目报告、作品展示... |
| **教学活动序列** | **第1-2课时**：引入与探索...<br>**第3-4课时**：概念建构与练习...<br>**第5-6课时**：应用与拓展...<br>**第7课时**：复习与评价... |
| **所需资源** | 教科书页码、PPT、学习单、教具（如圆规、计算器）、在线工具... |
| **差异化教学策略** | **对于学习困难学生**：提供脚手架（如公式提示卡）、简化任务...<br>**对于学有余力学生**：提供挑战性问题、开放性探究任务... |

### 1.2 课程设计自查清单（Curriculum Design Checklist）

在完成一个单元的设计后，可以用以下问题进行自查：

- [ ] 我的学习目标是否清晰、可衡量？
- [ ] 我的教学活动是否与学习目标紧密对齐？
- [ ] 我的评价方式能否真实反映学生是否达到了学习目标？
- [ ] 我是否为不同水平的学生都设计了有意义的学习机会？
- [ ] 我是否将数学知识与实际应用或学生的已有经验联系起来了？
- [ ] 课程内容是否在认知上具有挑战性，但又在学生的可接受范围内？
- [ ] 我是否为学生提供了"做数学"（论证、建模、交流）的机会，而不仅仅是"学数学"？

## 2. 教学活动设计工具

### 2.1 探究式学习（IBL）任务单框架

**主题**：____________________

1. **情境与问题 (Engage)**
    - *呈现一个故事、图片、视频或一个奇怪的现象，引出一个核心的数学问题。*
2. **猜想与假设 (Explore)**
    - "对于这个问题，你最初的想法是什么？"
    - "你觉得可能会有什么规律？"
    - *鼓励学生大胆猜测，不评判对错。*
3. **数据收集/实验验证 (Explain)**
    - "你需要哪些信息来验证你的猜想？"
    - "请设计一个实验或计算过程。"
    - *提供工具和数据，或引导学生自行收集。*
4. **模型建构/规律总结 (Elaborate)**
    - "你发现了什么规律？请用数学语言（公式、图表、文字）来描述它。"
5. **应用与反思 (Evaluate)**
    - "这个规律能用来解决什么新问题？"
    - "我们的结论在什么条件下成立？有没有例外？"

### 2.2 CPA教学法活动设计序列

**概念**：____________________

- **C - 具体操作 (Concrete)**：学生用**动手**的方式做什么？
  - *可使用物品：积木、计数棒、硬币、折纸、代数瓦...*
- **P - 图像表征 (Pictorial)**：学生用**画图**的方式做什么？
  - *可使用工具：模型图、条形图、数轴、示意图、流程图...*
- **A - 抽象符号 (Abstract)**：学生用**数学符号**做什么？
  - *如何将图像转化为数学表达式、方程或公式？*
  - *如何应用公式解决纯符号问题？*

### 2.3 数学游戏设计点子

- **"函数"版"你画我猜"**：一个同学拿到函数表达式，只能通过描述其性质（如"它经过原点"，"它在第一象限是增函数"）让另一个同学画出图像。
- **概率大富翁**：设计一个大富翁棋盘，每个格子的前进或后退由掷不同面数的骰子（4面、6面、8面）决定，让学生在游戏中直观感受不同事件的概率。
- **几何寻宝**：给出一系列几何指令（如"向东走5米，然后向北旋转90度走3米..."，涉及向量和坐标几何），让学生通过计算和测量找到"宝藏"。

## 3. 评价工具与量规

### 3.1 数学项目评价量规（Rubric）

| **维度** | **优秀 (4分)** | **良好 (3分)** | **合格 (2分)** | **需改进 (1分)** |
| :--- | :--- | :--- | :--- | :--- |
| **数学理解** | 对项目中涉及的数学概念有深刻、准确的理解，并能灵活运用。 | 对核心概念理解准确，但部分细节有误或不够深入。 | 对核心概念有基本理解，但存在一些重要错误。 | 对核心概念存在严重误解。 |
| **问题解决** | 成功地解决了核心问题，并能思考更复杂的情境。 | 成功地解决了核心问题，但过程可以更优化。 | 解决了问题的一部分，或在大量帮助下才解决。 | 未能解决问题。 |
| **数学表达** | 清晰、准确地使用了数学语言、图表和符号来呈现过程和结果。 | 能够使用数学语言表达，但不够规范或清晰。 | 尝试使用数学语言，但存在较多错误。 | 无法用数学语言进行表达。 |
| **协作与反思** | 在团队中扮演了积极角色，能清晰地阐述自己的贡献，并对项目进行了有深度的反思。 | 参与了团队合作，能够说明自己的工作，有基本的反思。 | 参与度有限，或无法清晰说明自己的工作。 | 没有参与合作或反思。 |

### 3.2 学生自我评价问卷

（在一个单元学习结束后）

- 在这个单元中，我最大的收获是：____________________
- 我觉得最困难/最有挑战性的部分是：____________________
- 对于____________这个概念，我（**）完全理解了，（**）基本理解，（__）还需要帮助。
- 为了学得更好，我下次可以尝试：____________________

## 4. 精选在线资源与软件

### 4.1 在线课程与标准

- **可汗学院 (Khan Academy)**: [https://www.khanacademy.org/](https://www.khanacademy.org/)
  - *提供系统化的K-12数学课程视频、练习和评测。*
- **美国共同核心州标准 (CCSS)官网**: [http://www.corestandards.org/Math/](http://www.corestandards.org/Math/)
  - *查阅最权威的CCSS数学内容和实践标准。*
- **NCTM (美国数学教师协会)**: [https://www.nctm.org/](https://www.nctm.org/)
  - *提供大量教学资源、期刊和专业发展机会。*

### 4.2 互动模拟与教具

- **GeoGebra**: [https://www.geogebra.org/](https://www.geogebra.org/)
  - *强大的数学"瑞士军刀"，集函数绘图、几何、代数、统计于一体的动态数学软件。*
- **Desmos**: [https://www.desmos.com/](https://www.desmos.com/)
  - *最好用的在线函数图形计算器，非常适合用于探索函数性质。*
- **Mathigon**: [https://mathigon.org/](https://mathigon.org/)
  - *"交互式数学教科书"，提供大量精美的互动课程和虚拟教具。*

### 4.3 竞赛与挑战平台

- **Brilliant.org**: [https://brilliant.org/](https://brilliant.org/)
  - *通过解决有趣、有挑战性的问题来学习数学和科学概念。*
- **美国数学竞赛 (AMC)**: [https://maa.org/math-competitions](https://maa.org/math-competitions)
  - *获取AMC 8/10/12等竞赛的历年真题和信息。*
